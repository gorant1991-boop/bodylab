- `index.html` — v4 (MVP), РАБОЧИЙ РЕФЕРЕНС. Не удалять, не ломать.
- `index_v5.html` — v5 (в разработке), широкий desktop-«пульт управления».

## Стек
- Чистый HTML/CSS/JS, без фреймворков и билда

- AI: Anthropic API (нужен прокси из-за CORS)

## v5 — архитектура модели (решено 2026-06-11)

### Модель времени: вероятностная (Монте-Карло), два режима
- «День»: одна жизнь, движок катит кости по шагам — повезло/не повезло.
- «Прогноз»: тот же движок × 1000 виртуальных жизней → вероятности на 1/5/10/20 лет
  (диабет 34%, инфаркт 12%, смерть 9%). Болезни КАСКАДЯТ (панкреатит→диабет→
  нефропатия→отказ почек; гипертония→инфаркт→быстрая смерть).

### Роль AI (НЕ в цикле Монте-Карло!)
1. Калибровка (1 запрос): профиль → JSON-матрица рисков под человека.
2. Движок (0 запросов): гоняет 1000 жизней на матрице — мгновенно, детерминированно.
3. Разбор (1 запрос): агрегат результатов → словесный анализ.
AI внутри MC-цикла ЗАПРЕЩЁН (тысячи запросов, латентность, недетерминизм).

### Три уровня детализации организма
1. Органы (кликабельные, на SVG): сердце, лёгкие, печень, почки, мозг, желудок,
   кишечник + НОВЫЕ: поджелудочная, сосуды, кожа. (10 шт)
2. Системы (индексы, без картинки): иммунитет, кости, эндокринная, кровь.
3. Биомаркеры (числа, для таблицы анализов): глюкоза, HbA1c, ЛПНП/ЛПВП, креатинин, АЛТ/АСТ.

### Орган = три числа (вместо одного)
`{base, reversible, permanent}`. Текущее = base − reversible − permanent.
reversible затухает со временем (полувыведение), permanent — необратимый рубец.

### Преднастройка профиля (стартовые условия)
хронические болезни (панкреатит, гипертония, диабет, астма, болезнь печени),
базовые витальные (рабочее давление, пульс покоя), образ жизни (часы сидячей
работы, стресс, сон), МЕСТО ПРОЖИВАНИЯ (Лазурный берег / Ямал — фоновый дрейф:
УФ, холод, полярная ночь → витамин D, нагрузка на сосуды).

## Фазы сборки v5
1. ✅ ГОТОВО. Модель организма: орган {base,rev,perm,seed}, +поджел/сосуды/кожа,
   системы-индексы, затухание, преднастройка→seed (не стирается при правке профиля),
   desktop-шелл, localStorage.
2. ✅ ГОТОВО. Граф болезней DISEASES (каскады: панкреатит→диабет→нефропатия→отказ почек;
   гипертония→ИБС→инфаркт; печень→цирроз→рак; +ХОБЛ, меланома). Движок: oneYear()→
   simulateLife()→computeForecast() (700 судеб). Острые события с летальностью.
   Привычки (курение/алкоголь/питание/активность) питают прогноз. «Прожить год» —
   живая прогрессия на S (урон→perm, нажитые болезни→profile.acquired).
   На экране «прогноз · 1000 судеб», клик по году = горизонт. Термин Монте-Карло — в коде.
3. ✅ ГОТОВО. Среда проживания: 7 регионов с факторами (uv/cold/pollution/altitude/
   polarNight/stressEnv/clean). deriveLifestyle читает factors; environmentDrift (годовой);
   checkOnset (смог→ХОБЛ, УФ→меланома); riskMult (cold→инсульт); applyPreconditions сеет
   стартовое (высота→O₂ база↓, полярная ночь→кости/иммунитет). Регионы дают разные судьбы:
   мегаполис→ХОБЛ, Ямал→гипертония, Лазурный→меланома.
   КАЛИБРОВКА: добавлена возрастная смертность (Гомпертц pAge=0.0004*e^((age-40)/9),
   усиление при avgOrg<70) + затухание урона (resist=0.4+0.6*org/100) + отказ органа
   вероятностный (<25%). Здоровый→85 даёт ~45% смерти (реалистично).
   ОСТАЁТСЯ ПОДКРУТИТЬ: умеренный курильщик в мегаполисе ~70%@20 — на верхней границе.
4. AI оживает (калибровка матрицы рисков + разбор). Заглушка во вкладке AI.
   Идея: AI может помочь с калибровкой чисел графа под реальную эпид-статистику.
5. Таблица анализов → биомаркеры (режим для специалистов).

## Архитектура движка (Фаза 2)
- DISEASES — граф: yearlyDamage, transitions[{to,base,mods}], acute{base,lethality,mods}.
- deriveLifestyle(profile) → {smoking,alcohol,diet,activity,obese,uv,...}; riskMult(mods,L).
- oneYear(org,vit,active,L,age) — мутирует, возвращает {death,news}. Ядро, общее для:
  simulateLife (клон, read-only от S) и stepLiveYear (мутирует S).
- forecast кэшируется; invalidate() сбрасывает при изменениях (доза, год, профиль).
- ВАЖНО: AI в цикл Монте-Карло НЕ ставить (см. выше).

## Важно
